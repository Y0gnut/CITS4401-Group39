
#### 1. User (用户中心)

- **设计原因**：作为系统架构的“Actor Hub”。
    
- **深层逻辑**：不仅是为了存账号，而是为了实现 **RBAC (角色权限控制)**。通过统一的 User 实体，我们将系统内的所有动作（如审批、指派、声明冲突）与具体身份绑定。
    
- **关键点**：这满足了审计溯源中“谁 (ActorID) 做了什么 (Action)”的必要前提。
    

#### 2. Application (核心工作流实体)

- **设计原因**：作为“Aggregate Root (聚合根)”。
    
- **深层逻辑**：申请不仅是数据记录，更是一个**状态机 (State Machine)**。通过 ParentApplicationID 的自引用，我们将复杂的 Amendment/Extension 业务平铺化。
    
- **关键点**：避免了为“修订申请”建立额外实体的代码复杂度，通过 RevisionDeadline 字段与后台定时器联动，实现自动逾期提醒。
    

#### 3. Assignment (指派控制台)

- **设计原因**：业务流程的“隔离层”。
    
- **深层逻辑**：如果不设此实体，评审决策将直接挂在 Application 上，导致“多人协作”逻辑混乱。Assignment 允许系统精确追踪“哪个成员对应哪个指派任务”。
    
- **关键点**：它允许 Chair 对任务进行重分配，而不会影响 Application 的原始业务数据。
    

#### 4. ReviewDecision (评审决策闭环)

- **设计原因**：系统“再评审循环”的信号发射器。
    
- **深层逻辑**：评审是一个循环过程，而非单一行为。通过 is_submitted 状态位，我们建立了一个**“软锁定”机制**——一旦文档更新，即重置该字段，强制评审员再次评估。
    
- **关键点**：这是解决“评审员审批了旧版本文档”这一合规隐患的技术核心。
    

#### 5. Annotation (颗粒度溯源)

- **设计原因**：将反馈锚定到文档的特定版本。
    
- **深层逻辑**：仅在 Application 层面记录意见是无效的，因为文档在变。关联 VersionID 确保评审员的每一条建议都对应于研究人员提交的特定二进制快照。
    
- **关键点**：支撑了 REQ-REV-04，确保反馈精确到文档的段落，减少沟通歧义。
    

#### 6. ConflictDeclaration (合规拦截哨兵)

- **设计原因**：利益冲突的强制披露与处理闭环。
    
- **深层逻辑**：不将冲突声明写在 Annotation 里，是因为冲突是关于**“权限”**而非**“内容”**的。通过独立实体，系统可以实现业务拦截逻辑（即：有Pending声明时，Assignment 无法获得决策权）。
    
- **关键点**：这是评审环节的“门禁系统”，直接关系到系统评审的公正性和合规性。
    

#### 7. Document & DocumentVersion (版本控制真相)

- **设计原因**：实现“单一真相来源 (Single Source of Truth)”。
    
- **深层逻辑**：Document 保持文档命名逻辑的唯一性，DocumentVersion 管理具体的二进制文件变迁。这使得 side-by-side 的对比成为可能。
    
- **关键点**：极大减轻了 Application 组件的 IO 压力，符合 REQ-US-01 的性能约束。
    

#### 8. Communication (行政转译缓冲区)

- **设计原因**：维护行政人员 (Admin) 与研究人员 (PI) 的沟通通道。
    
- **深层逻辑**：这是伦理管理中非常关键的**中间处理层**。评审员意见不直接发送，而是经由 Admin 整合，记录在 Communication 实体中，这确保了行政发布的口径统一且有据可查。
    
- **关键点**：作为行政 Buffer，它拦截了评审员反馈，减少了 PI 直接与评审员对峙带来的潜在矛盾。
    

#### 9. AuditEntry (forensic 日志)

- **设计原因**：合规性审查的底线。
    
- **深层逻辑**：该表不处理业务逻辑，它是系统的**“黑匣子”**。任何实体的状态变化（Status Update）和决策发布（Decision Submission）必须同步插入此表。
    
- **关键点**：在面对行业审计时，这是证明系统流程合规的法律凭证。
